Decision Framework
When facing any implementation choice, ALWAYS ask in this order:

Does this serve the core loop? (campaign → response → AI ranking → founder pays respondent)
Does this protect quality? (anti-gaming, anti-low-effort)
Does this feel right for the target user? (18–32, direct tone, no corporate feel)
Is this the simplest version that works?

If the answer to any is no, reconsider.

Here's your build order (one thing at a time) you will ask me before proceeding with every new section if ok to proceed):

Landing Page — marketing page, value prop, how it works, pricing, waitlist CTA
Auth & Onboarding — sign up/login, phone verification, role selection (founder or respondent), basic profile

Founder Dashboard — active campaigns, past campaigns, responses received, money allocated, empty states
Campaign Builder — freeform notepad → AI questions → Signal Strength Meter → standard MC questions → bonus pool setup → launch

Respondent Dashboard — available campaigns, past responses, earnings, pending payouts
Response Flow — sequential questions, paste disabled, MC interspersed, later questions reference earlier answers, submit
AI Response Ranking — quality scoring, MC consistency checks, ranked list presented to founder
Founder Review & Payout Allocation — founder reads ranked responses, allocates bonus pool to chosen respondents
Payment System — bonus distribution, 15% platform cut, payout thresholds, billing for subscription tiers and overage fees
Anti-Gaming Layer — behavioral tracking, device fingerprinting, IP clustering, reputation scoring, community flagging
Subscription Management — free/starter/pro/scale tiers, overage handling, upgrade/downgrade flows
